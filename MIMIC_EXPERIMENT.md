# Style-Mimicking LLM vs. Human Post-Editing — Experiment Design

This document describes a follow-up experiment that goes **beyond** Baumler et
al. (ACL 26) by adding two new conditions:

- `gpt-5.5-mimic`: a draft generated by GPT-5.5 *while looking at* the
  participant's own unassisted (control) writing as a style demonstration.
- `claude-opus-4-7-mimic`: same idea, with Claude Opus 4.7.

We then compare these to the paper's two existing conditions:

- `LLM-original`: the unconditioned GPT-o4-mini draft used in the study.
- `human-edited`: the participant's post-edited final text.

**Hypothesis.** Modern frontier LLMs, given a few unassisted writing samples
from the author, can match the author's personal style at least as well as a
human post-editor can on top of an o4-mini draft.

## 1. Procedure

For each of the 324 treatment tasks in the released logs:

1. Read the participant's planning `details`.
2. Read both of that participant's `control` texts (unassisted writing on the
   *other* tasks they completed) — this is the only style information the model
   sees about that author.
3. Build the prompt below and call the target model. Generation is constrained
   to a single completion with `reasoning.effort=medium` (OpenAI) or default
   adaptive thinking (Anthropic), `max_tokens` set to leave room for a ≈150-word
   response.
4. Cache the generated text on disk keyed by SHA-256 of (model name, pid,
   task_idx, scenario, details, samples) so the run is idempotent and partial
   failures are recoverable.

### 1.1 Prompt

```
{instruction}  Only respond with the draft with no additional content or
explanation. Please do not include any placeholder text such as "[name]". The
final draft should be about 150 words.

Here are some details to help you write:
{details}

Below are two short writing samples written by the same author, on different
topics, in their natural unassisted style. Mimic this author's personal writing
style as closely as possible -- their typical sentence length, punctuation
habits, vocabulary register, and tone:

--- STYLE SAMPLE 1 ---
{control_text_1}

--- STYLE SAMPLE 2 ---
{control_text_2}
```

`{instruction}` is the same per-scenario instruction used by Baumler et al.
(verbatim for "wedding"; the same template applied to the eight scenarios
actually used in the study).

`{details}` is the participant-written planning text already stored in the log.

The two `{control_text_*}` are taken **only from the participant's own control
tasks**, so this is a within-author test: each model sees only that author's
writing, and generates a draft for a *different* task than the samples cover
(scenarios are randomised so the demos are guaranteed not to overlap with the
target task).

### 1.2 Metric

Following the paper, we compute LUAR-MUD cosine similarity between the
generated draft and that participant's own control style centroid:

  `s_self(approach, task) = cos( embed(draft), centroid( control texts of pid ) )`

with **the participant's own control texts excluded if the draft itself is
based on them**. (For mimic drafts this is automatic — we use *all* of the
participant's controls as the centroid, but we computed the embedding from a
generated draft, not from the controls themselves.)

We also compute `sim_mimic_control_other_mean` (mean cos to other participants'
control-pool style centroids) so we can check whether the model is actually
targeting *this* author rather than just producing generically humanish text.

### 1.3 Statistical tests

For each generator G we run three paired permutation tests (per task,
n_perm = 10 000), with Hedges' *g* (pooled SD) and 95 % bootstrap CIs:

| Test | x | y | If `g > 0` then ... |
|---|---|---|---|
| G vs. LLM | `s_self(G)` | `s_self(LLM-original)` | G is closer to author than the unconditioned LLM draft |
| G vs. human-edited | `s_self(G)` | `s_self(human-edited)` | **G beats human post-editing on style fidelity** |
| G self vs. G other | `s_self(G)` | `s_other(G)` | G is targeting *this* author, not just being humanish |

## 2. How to run

### 2.1 One-time setup

The experiment needs both API keys. Add them as **secrets** in the Cursor
Dashboard (Cloud Agents → Secrets):

- `OPENAI_API_KEY` (for `gpt-5.5`)
- `ANTHROPIC_API_KEY` (for `claude-opus-4-7`)

Secrets persist across cloud-agent runs and are injected as environment
variables, so the next agent run will pick them up automatically.

### 2.2 Generation

```
make mimics            # both models, 324 tasks each (cached, idempotent)
# or per-model:
python scripts/07_generate_mimics.py --generators gpt-5.5
python scripts/07_generate_mimics.py --generators claude-opus-4-7
# small smoke test without API keys:
python scripts/07_generate_mimics.py --generators stub --limit 50
```

### 2.3 Comparison

```
make mimic-compare     # embeds drafts with LUAR, builds Fig. 9, writes results/mimic_comparison.csv
```

`scripts/08_compare_mimics.py` is robust to partial caches: you can run
generation halfway through, then `make mimic-compare` to get an interim picture
on whatever fraction of the 324 tasks have completed so far.

## 3. Expected costs (rough order-of-magnitude)

Per task, each prompt is roughly 600 input tokens (instruction + details + two
control samples) and ≈ 200 output tokens. Across 324 tasks × 2 models that
gives ~ 0.4 M input tokens and ~ 0.13 M output tokens total.

At advertised list prices (~$5/MTok input, ~$25-30/MTok output for both models)
that's roughly **$5–$8 in API spend** for the full pre-registered run. Caching
makes reruns free.
